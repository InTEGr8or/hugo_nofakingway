Choosing Hugo Over DocPad

I love the flexibility of static generated sites. I can create content easily while I also develop the logical structure of the components and their taxonomical relations. Most of that is provided by Markdown, including the YAML frontmatter. So the static generator just needs to be easy to start and extend, and it also needs to be sustainable. It doesn’t make sense to invest in learning how to use a static site generator that is not being actively used by a comunity of people.

Developing in Docpad

I’ve started developing in Docpad because I love lo-fi text-only documentation. This CMS uses folder structure and tags for site hierarchy and is very flexible. The MarkDown is transpiled into html and I can use a veriety of templating languages for the layout components. I’m going under the assumption that this will all provide maximal flexability while I am actively creating content.
